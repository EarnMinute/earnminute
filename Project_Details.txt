# PROJECT NAME

**EarnMinute**

---

# PROJECT TYPE

Full-stack **freelance micro-task marketplace platform**.

Concept inspiration:

* Fiverr
* Upwork
* Micro-task marketplaces

The system allows employers to post small tasks and freelancers to complete them quickly for payment.

# PROJECT ARCHITECTURE

Backend uses **Layered Architecture**

```
Controller → Service → Repository → Database
```

Folder structure:

```
backend/

controllers/
services/
repositories/
models/
routes/
middleware/

server.js
```

### Responsibilities

Controllers
Handle HTTP requests and responses.

Services
Contain business logic.

Repositories
Handle database queries.

Models
Define MongoDB schemas.


# IMPORTANT DEVELOPMENT RULES

Before modifying code:

1. Analyze the full codebase
2. Ask for the relevant file
3. Return the **FULL updated file** Unless it is bigger than ~200 lines. If it is bigger than ~200 lines then you can give the code snippet and the exact details of where to paste the code (after what and before what).
4. Avoid breaking existing features (very important)
5. Maintain architecture

```
Controller → Service → Repository
```

Backward compatibility must be preserved.
