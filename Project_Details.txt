# EarnMinute — Project Details

## Project Overview

**EarnMinute** is a full-stack **micro-task freelance marketplace platform** where employers can post small tasks and freelancers can apply, complete work, and receive payment.

The platform focuses on **short tasks that can be completed quickly**, enabling users to earn money by completing small freelance jobs.

Typical tasks may include:

* Data entry
* Social media tasks
* Simple design work
* Writing small content pieces
* Online research
* Website testing
* File conversions

The platform is designed with **Bangladesh-focused payment systems** and will later support:

* bKash
* Nagad
* SSLCommerz
* Internal wallet system

---

# Core Platform Goals

The system is designed to prioritize:

* Clean architecture

# Task Lifecycle (State Machine)

The platform enforces a **strict task lifecycle** to prevent invalid actions.

Task states:

```
open
assigned
in_progress
submitted
revision_requested
approved
completed
cancelled
disputed
```

Lifecycle flow:

```
open
 ↓
assigned
 ↓
in_progress
 ↓
submitted
 ↓
revision_requested (optional)
 ↓
approved
 ↓
completed
```

Alternative paths:

```
open → cancelled
assigned → cancelled
submitted → disputed
```

Rules include:

* freelancer must start task before submission
* employer cannot cancel after work begins
* submission required before approval
* escrow must be funded before work begins

---

# Escrow Workflow

Escrow ensures secure payments.

Workflow:

```
1 Employer posts task
2 Employer funds escrow
3 Freelancer starts task
4 Freelancer submits work
5 Employer reviews submission
6 Employer approves work
7 Escrow releases payment
8 Task completes
```

If dispute occurs:

```
Escrow held until admin resolution
```

# Frontend Task Engine

The frontend uses a **Task Action Engine** that determines which actions are available based on:

```
task status
user role
```

Example:

```
assigned → freelancer can start task
in_progress → freelancer can submit work
submitted → employer can approve or request revision
approved → employer can complete task
```

This ensures the UI always reflects the backend lifecycle.
